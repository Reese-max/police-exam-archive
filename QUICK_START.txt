================================================================================
            警察考古題 Google Forms 自動部署 - 快速開始指南
================================================================================

【最快速的方式】

  雙擊執行「一鍵部署.bat」，然後選擇功能即可！

================================================================================

【功能選單】

  [1] 完整部署 - 從 CSV 到表單一條龍完成
  [2] 只驗證現有表單 - 檢查所有表單狀態
  [3] 只推送程式碼 - 更新 Apps Script
  [4] 查看表單清單 - 列出所有已建立的表單
  [5] 擷取 CLASP 日誌 - 下載執行記錄
  [Q] 退出

================================================================================

【完整部署流程 (選擇 1)】

  Step 1: 選擇要處理的 CSV 檔案
  Step 2: 自動生成 Apps Script 程式碼
  Step 3: 自動推送到 Google
  Step 4: 開啟瀏覽器，你點擊「執行」建立表單
  Step 5: 開啟驗證頁面，你點擊「執行」驗證
  Step 6: 自動產生 QA 報告

================================================================================

【首次使用前的設定】

  1. 確認已安裝 Python 3.9+
  2. 安裝 Python 套件（僅測試用）：pip install -r requirements.txt
  3. 確認已安裝 Node.js
  4. 安裝 CLASP: npm install -g @google/clasp
  5. 登入 CLASP: clasp login
  6. 在 GCP 啟用 Drive API 和 Forms API

================================================================================

【檔案說明】

  一鍵部署.bat          - 雙擊啟動智能部署系統
  smart_deploy.py       - 智能自動化腳本
  auto_generate_form.py - CSV 解析與程式碼生成
  src/Code.gs           - 表單建立腳本
  src/FormValidator.gs  - 表單驗證腳本
  form_generation_summary_*.json - 匯入報告
  QA_Report_*.txt       - QA 驗證報告

【管線備註】
  - 推薦使用根目錄的 `.clasp.json`（scriptId: 1-m81…），對應 `src/Code.gs` + `FormValidator.gs`。
  - `考選部考古題完整庫/情境實務考古題/.clasp.json` 為舊專案設定，除非要維護舊表單，否則不要在該子資料夾跑 clasp push/run。

================================================================================

【常見問題】

  Q: clasp push 失敗？
  A: 執行 clasp login 重新登入

  Q: validateAllForms 出錯？
  A: 確認已在 GCP Console 啟用 Drive API 和 Forms API

  Q: 出現 403 access_denied？
  A: 在 GCP OAuth 同意畫面加入你的 email 為測試使用者
